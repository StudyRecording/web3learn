# 1.1 Web 的演进

## Web 1.0：只读时代 (1990-2004)

### 特征

- **静态内容**：HTML 页面，很少更新
- **单向传播**：网站发布内容，用户被动阅读
- **中心化托管**：内容由网站所有者控制

### 技术栈

```
Web 1.0 技术栈
├── 前端：HTML + CSS (少量)
├── 后端：CGI, Perl, PHP
├── 数据库：文件系统为主
└── 服务器：Apache
```

### 代表产品

- 门户网站：Yahoo, 新浪
- 个人主页
- 论坛 (BBS)

## Web 2.0：读写时代 (2004-至今)

### 特征

- **用户生成内容 (UGC)**：用户创造并分享内容
- **社交互动**：评论、点赞、分享
- **平台化**：巨头提供平台，用户贡献数据
- **商业模式**：免费服务 + 广告变现

### 技术栈

```
Web 2.0 技术栈
├── 前端：React/Vue/Angular
├── 后端：Java Spring/Node.js/Go
├── 数据库：MySQL/MongoDB/Redis
├── 云服务：AWS/阿里云/GCP
└── 移动端：iOS/Android
```

### 代表产品

- 社交媒体：Facebook, Twitter, 微信
- 内容平台：YouTube, 抖音, 小红书
- 协作工具：Google Docs, 飞书

### Web 2.0 的问题

```
┌─────────────────────────────────────────────────────────┐
│               Web 2.0 的核心问题                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 数据所有权问题                                       │
│     ├── 用户数据存储在平台服务器                         │
│     ├── 平台可以删除账号、封禁用户                       │
│     └── 用户无法真正"拥有"自己的数据                     │
│                                                         │
│  2. 平台垄断问题                                         │
│     ├── 少数巨头控制流量和内容分发                       │
│     ├── 算法决定用户看到什么                             │
│     └── 平台政策可能随时改变                             │
│                                                         │
│  3. 隐私安全问题                                         │
│     ├── 数据泄露事件频发                                 │
│     ├── 用户数据被商业化利用                             │
│     └── 缺乏真正的隐私保护                               │
│                                                         │
│  4. 价值分配问题                                         │
│     ├── 创作者获得的价值比例低                           │
│     ├── 平台抽成过高                                     │
│     └── 用户贡献无法获得应有回报                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 案例：游戏内购

```java
// Web 2.0 游戏内购模型
// 用户购买道具，但所有权归属游戏公司

class GameAccount {
    private String userId;
    private List<GameItem> items;  // 存储在游戏公司服务器
    
    public void purchaseItem(GameItem item) {
        // 用户付费购买
        items.add(item);
        // 但如果游戏公司：
        // - 停止运营
        // - 封禁账号
        // - 修改规则
        // 用户将失去所有道具
    }
}
```

## Web 3.0：读、写、拥有时代 (2014-未来)

### 概念提出

2014年，以太坊联合创始人 Gavin Wood 首次提出 Web 3.0 概念，核心思想：

> Web 3.0 是一个去中心化的网络，用户真正拥有自己的数据和数字资产。

### 核心特征

```
Web 3.0 特征
├── 去中心化
│   ├── 无单点故障
│   ├── 无中心化控制
│   └── 数据分布在网络节点
│
├── 用户主权
│   ├── 用户拥有数据
│   ├── 用户拥有身份
│   └── 用户拥有资产
│
├── 原生经济
│   ├── 内置支付系统
│   ├── 代币激励机制
│   └── 价值自由流动
│
└── 开放透明
    ├── 代码开源
    ├── 规则透明
    └── 可验证可审计
```

### 技术栈

```
Web 3.0 技术栈
├── 智能合约：Solidity/Vyper/Rust
├── 区块链：Ethereum/Solana/Polygon
├── 存储：IPFS/Arweave
├── 前端：React/Vue + ethers.js
├── 身份：钱包地址/DID
└── 索引：The Graph
```

### Web 3.0 解决的问题

| 问题 | Web 2.0 解决方案 | Web 3.0 解决方案 |
|------|-----------------|-----------------|
| 数据所有权 | 存储在平台 | 存储在链上，用户拥有 |
| 平台封禁 | 无法申诉 | 代码即法律，无人能封禁 |
| 支付 | 需要第三方支付 | 原生加密货币支付 |
| 身份 | 平台账号 | 钱包地址，自主控制 |
| 隐私 | 依赖平台保护 | 密码学保护 |

### Web 3.0 案例：NFT 游戏

```solidity
// Web 3.0 游戏道具 NFT
// 用户真正拥有道具，可以自由交易

contract GameItem is ERC721 {
    // 道具作为 NFT 存储在链上
    // 用户钱包地址是真正的所有者
    
    function mintItem(address player) public {
        _mint(player, totalSupply());
        // 一旦铸造，只有用户能转移
        // 游戏公司无法回收
    }
    
    // 用户可以：
    // 1. 在任何市场出售
    // 2. 转移到其他游戏
    // 3. 永久保存（即使游戏停止运营）
}
```

```
┌──────────────────────────────────────────────────────────────┐
│                  Web 演进对比                                 │
├──────────────┬───────────────┬───────────────┬───────────────┤
│     维度      │    Web 1.0    │    Web 2.0    │    Web 3.0    │

```
