## 核心要点

### Web 演进三阶段

```
┌─────────────────────────────────────────────────────────────┐
│                    Web 演进历程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Web 1.0 (1990-2004)                                        │
│  ├── 特点：只读                                              │
│  ├── 内容：静态网页                                          │
│  └── 用户：被动消费                                          │
│                                                             │
│  Web 2.0 (2004-至今)                                         │
│  ├── 特点：读写                                              │
│  ├── 内容：用户生成 (UGC)                                    │
│  ├── 用户：参与互动                                          │
│  └── 问题：数据被平台垄断                                    │
│                                                             │
│  Web 3.0 (2014-未来)                                         │
│  ├── 特点：读、写、拥有                                      │
│  ├── 内容：用户拥有                                          │
│  ├── 用户：真正的主人                                        │
│  └── 核心：去中心化                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Web3 核心原则

1. **去中心化** - 不由单一实体控制
2. **无需许可** - 任何人都能参与
3. **原生支付** - 内置加密货币支付
4. **去信任** - 通过代码而非第三方建立信任

### 从 Java 开发者视角看 Web3

| 概念 | Web2 (Java) | Web3 |
|------|-------------|------|
| 后端服务 | Spring Boot 应用 | 智能合约 |
| 数据库 | MySQL/PostgreSQL | 区块链状态 |
| 用户认证 | JWT/Session | 钱包签名 |
| API | REST/GraphQL | 合约方法调用 |
| 部署 | 云服务器 (AWS/阿里云) | 区块链网络 |
| 代码修改 | 随时更新 | 不可变 (需代理模式) |
